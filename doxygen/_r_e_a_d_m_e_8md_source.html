3 NEURON is a simulator for models of neurons and networks of neuron.
4 See [http://neuron.yale.edu](http://neuron.yale.edu) for installers, source code,
5 documentation, tutorials, announcements of courses and conferences,
6 and a discussion forum.
7 
8 ## Building NEURON
9 
10 NEURON provided binary installers for Linux, Mac and Windows platform [here](https://neuron.yale.edu/ftp/neuron/versions/alpha/). If you want to build latest version from source, you can find instructions bellow.
11 
12 ### Build Dependencies
13 
14 In order to build NEURON from source, following packages must be available:
15 
16 - Bison
17 - Flex
18 - C/C++ compiler suite
19 
20 Following packages are optional (see build options):
21 
22 - Python >=2.7, or Python >=3.5 (for Python interface)
23 - Cython (for RXD)
24 - MPI (for parallel)
25 - X11 (Linux) or XQuartz (MacOS) (for GUI)
26 
27 ### Build using Autotools
28 
29 If you would like to have GUI support, you first need to install Interviews package available from GitHub [here](http://github.com/neuronsimulator/iv) or tarball provided [here](http://neuron.yale.edu/ftp/neuron/versions/alpha/). In case of git repository, first you need to run `build.sh` script to create the automake, autoconf, libtool generated files:
30 
31 ```bash
32 sh build.sh
33 ```
34 
35 And then run standard `configure`, `make` and `make install` steps to install Interviews:
36 
37 ```bash
38 ./configure
39 make
40 make install
41 ```
42 
43 To build NEURON we have to use same steps as Interviews i.e. if source is obtained from the git repository, run `build.sh` script to create the automake, autoconf, libtool generated files:
44 
45 ```bash
46 sh build.sh
47 ```
48 
49 and then run standard `configure`, `make` and `make install` steps:
50 
51 ```bash
52 ./configure
53 make
54 make install
55 ```
56 
57 If you want to customize build, particularly useful configure options are:
58 
59 
60 - `--prefix=/some/path` : Install in this location of your filesystem.
61 - `--without-x` : If the InterViews graphics library is not installed, disable GUI.
62 - `--with-iv=<prefix>/../iv` : If InterViews was not installed in <prefix>/../iv
63 - `--with-paranrn` : Parallel models on cluster computers using MPI
64 - `--with-nrnpython` : Use Python as an alternative interpreter (as well as the native HOC interpreter).
65 - `--with-nmodl-only` : Build nmodl only (in case of cross compiling)
66 - `--disable-rx3d` : Do not compile the cython translated 3-d rxd features
67 
68 
69 For more details see the [INSTALL.md](https://github.com/neuronsimulator/nrn/blob/master/INSTALL.md)
70 file.
71 
72 For more installation information see: [https://neuron.yale.edu/neuron/download/getdevel](https://neuron.yale.edu/neuron/download/getdevel).

74 ### Build using CMake
75 
76 NEURON can now also be built and installed using [CMake build system](https://cmake.org/). Currently we are supporting two build systems i.e. Autoconf and CMake. In the future, based on the feedback from the community we will decide on switching to CMake build system. Therefore, we would be grateful for any feedback or issues you encounter using CMake based build system. Please [report an issue here](https://github.com/neuronsimulator/nrn/issues) and we will be happy to help.
77 
78 One of the primary advantage of CMake based build system is integration with other projects like [Interviews](https://github.com/neuronsimulator/iv), [CoreNEURON](https://github.com/BlueBrain/CoreNeuron/), [NMODL](https://github.com/BlueBrain/nmodl/) etc. Such projects are now integrated into single CMake based build system and they can be installed together as shown below:
79 
80 
81 1. Clone latest version:
82 
83  ```
84  git clone https://github.com/neuronsimulator/nrn
85  cd nrn
86  ```
87 
88 2. Create a build directory:
89 
90  ```
91  mkdir build
92  cd build
93  ```
94 
95 3. Run cmake with the appropriate options (see below for list of common options). \
96 A full list of options can be found in *nrn/CMakeLists.txt* . Defaults are shown in *nrn/cmake/BuildOptionDefaults.cmake*), \
97 e.g. a bare-bones install:
98 
99  ```
100  cmake .. \
101  -DNRN_ENABLE_INTERVIEWS=OFF \
102  -DNRN_ENABLE_MPI=OFF \
103  -DNRN_ENABLE_RX3D=OFF
104  ```
105 
106 Note that if you are building on Cray system with GNU toolchain, you have to set following environmental variable **before** calling above CMake comamnd:
107 
108 ```
109 export CRAYPE_LINK_TYPE=dynamic
110 ```
111 
112 4. Build the code:
113 
114  ```
115  make -j
116  make install
117  ```
118 
119 Particularly useful CMake options are (use **ON** to enable and **OFF** to disable feature):
120 
121 * **-DNRN\_ENABLE\_BINARY_SPECIAL=ON** : Build special as a binary instead of shell script
122 * **-DNRN\_ENABLE\_INTERVIEWS=OFF** : Disable Interviews (native GUI support)
123 * **-DNRN\_ENABLE\_PYTHON=OFF** : Disable Python support
124 * **-DNRN\_ENABLE\_MPI=OFF** : Disable MPI support for parallelization
125 * **-DNRN\_ENABLE\_RX3D=OFF** : Disable rx3d support
126 * **-DNRN\_ENABLE\_CORENEURON=ON** : Enable CoreNEURON support
127 * **-DNRN\_ENABLE\_TESTS=ON** : Enable unit tests
128 * **-DPYTHON\_EXECUTABLE=/python/binary/path** : Use provided Python binary to build Python interface
129 * **-DCMAKE_INSTALL_PREFIX=/install/dir/path** : Location for installing
130 * **-DCORENRN\_ENABLE\_NMODL=ON** : Use [NMODL](https://github.com/BlueBrain/nmodl/) instead of [MOD2C](https://github.com/BlueBrain/mod2c/) for code generation with CoreNEURON
131 
132 Please refer to [docs/cmake_doc/options.rst](docs/cmake_doc/options.rst) for more information on the CMake options.
