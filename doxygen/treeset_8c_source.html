83 (2002-04-06)
84 One reason for going into the following in depth is to allow
85 comparison of the fixed step implicit method
86 with the implementation of the corresponding
87 cvode (vext not allowed) and daspk (vext allowed) versions.
88 
89 In the fixed step method the result of a setup is a linear matrix
90 equation of the form
91  G*(vinew,vxnew) = i(vm,vx)
92 which is solved and followed by an update step
93  vx=vxnew
94  vm = vinew - vxnew
95 
96 When cvode was added, as much as possible of the existing setup was re-used
97 through the addition of flags. Cvode solves a system of the form
98  dy/dt = f(y)
99 and this is not obtainable in the presence of extracellular or LinearMechanism
100 instances. And even in the cases it can handle, it
101 required a good deal of manipulation to remove the zero area compartments
102 from the state vector and compute the algebraic equations separately.
103 In fact, as of 2002-04-06, the CONSERVE statement for kinetic schemes
104 in model descriptions is still ignored.
105 At any rate, the issues involved are
106 1) How to compute f(y). Any algebraic equations are solved inside f(y)
107 so that the algebraic states are consistent with y.
108 2) How to setup the jacobian (I - gamma*df/dy)*y = b
109 
110 When daspk was added which solves the form
111  0 = f(y, y')
112 having one implementation serving the setup for
113 three very distinct methods became too cumbersome. Therefore setup_tree_matrix
114 was copied twice and modified separately so that each method
115 had its own setup code derived from this file.
116 
117 The issues that have
118 arisen with daspk are
119 1) How to actually implement a residual function res = f(y, y')
120 2) How to actually implement a preconditioner setup of the form
121  (df/dy + cj*df/dy')*y = b
122 3) how to initialize so that 0 = f(y(0), y'(0))
123 when it is not possible to know which are the algebraic and which
124 are the differential equations. Also, it is often the case that
125 a linear combination
126 of apparently differential equations yields an algebraic equation and we
127 don't know the linearly dependent subsets.
128 DASPK itself will only handle initialization if one can mark some states
129 as differential and the remaining states as algebraic.
130 
131 With that, here's what is going on in this file with respect to setup of
132  G*(vinew, vxnew) = i(vm,vx)
133 
134 The current balance equations are sum of outward currents = sum of inward currents
135 For an internal node
136  cm*dvm/dt + i(vm) = is(vi) + ai_j*(vi_j - vi)
137 For an external node (one layer)
138  cx*dvx/dt + gx*(vx - ex) = cm*dvm/dt + i(vm) + ax_j*(vx_j - vx)
139 where vm = vi - vx
140 and every term has the dimensions mA/cm2
141 (the last terms in each equation represent the second derivative cable properties)
142 The chosen convention that determines the side of the terms and their sign
143 is that
144 positive membrane current is outward but electrode stimulus
145 currents are toward the node. We place the terms involving
146 currents leaving a node on the left, and currents entering a node
147 on the right. i.e.
148  |is
149  ia `.'
150  ---> <---
151  |im
152  ie `.'
153  ---> <---
154  |gx*vx
155  `.'
156  ---
157  \ /
158  -
159 
160 At the hoc level, v refers to vm and vext refers to vx.
161 It should also be realized that LinearMechanism instances overlay equations of
162 the form
163 c*dy/dt + g*y = b
164 onto the current balance equation set thus adding new states,
165 equations, and terms to the above equations. However,
166 note that when y refers to a voltage of a segment in those equations
167 it always means vi or vx and never vm.
168 
169 At this point it is up to us whether we setup and solve the form
170 G*(vm,vx) = i or G*(vi,vx) = i
171 since any particular choice is resolved back to (vm,vx) in the update step.
172 At present, particularly for the implicit fixed step method whose setup
173 implementation resides in this file, we've chosen G*(vi,vx) = i,
174 because there are fewer operations in setting up second derivatives and it
175 makes life easy for the LinearMechanism. i.e. on setup each element of the
176 linear mechanism gets added to only one element of the complete matrix equations
177 and on update we merely copy the
178 vi solution to the proper LinearMechanism states and update v=vi-vext
179 for the compartment membrane potentials.
180 
181 The implicit linear form for the compartment internal node is
182 cm/dt*(vmnew - vmold) + (di/dvm*(vmnew - vmold) + i(vmold))
183  = (dis/dvi*(vinew - viold) + is(viold)) + ai_j*(vinew_j - vinew)
184 and for the compartment external node is
185 cx/dt*(vxnew - vxold) - gx(vxnew - ex)
186  = cm/dt*(vmnew - vmold) + (di/dvm*(vmnew - vmold) + i(vmold))
187  + ax_j*(vxnew_j - vxnew) = 0
188 
189 and this forms the matrix equation row (for an internal node)
190  (cm/dt + di/dvm - dis/dvi + ai_j)*vinew
191  - ai_j*vinew_j
192  -(cm/dt + di/dvm)*vxnew
193  =
194  cm/dt*vmold + di/dvm*vmold - i(vmold) - dis/dvi*viold + is(viold)
195 
196 where old is present value of vm, vi, or vx and new will be found by solving
197 this matrix equation (which is of the form G*v = rhs)
198 The matrix equation row for the extracellular node is
199  (cx/dt + gx + ax_j + cm/dt + di/dvm)*vext
200  -ax_j*vxnew_j
201  -(cm/dt + di/dvm)*vinew
202  = cx/dt*vxold - (cm/dt*vmold + di/dv*vmold - i(vmold)) - gx*ex
203 
204 Node.v is always interpreted as membrane potential and it is important to
205 distinguish this from internal potential (Node.v + Node.extnode.v)
206 when the extracellular mechanism is present.
207 
208 In dealing with these current balance equations, all three methods
209 use the same mechanism calls to calculate di/dvm, i(vm), dis/dvi, is(vi)
210 and add them to the matrix equation.
211 Thus the passive channel, i=g*(v - e), is a transmembrane current
212 (convention is positive outward) and the mechanism adds g to the
213 diagonal element of the matrix and subtracts i - g*v from the right hand side.
214 (see nrn_cur of passive.c) It does NOT add anything to the extracellular
215 current balance row.
216 
217 An electrode current (convention positive current increases the internal potential)
218 such as a voltage clamp with small resistance, is=g*(vc - (v+vext))
219 (see nrn_cur of svclmp.c)
220 subtracts -g from the matrix diagonal (since dis/dvi = -g)
221 and adds i - g*(v+vext) to the
222 right hand side. In the presence of an extracellular mechanism, such electrode
223 mechanisms add the dis/dvi term to the extracellular equation diagonal and also add
224 is - dis/dvi*(viold) to the extracellular equation right hand side.
225 
226 This is where a trick comes in. Note that in the equations it is the
227 membrane currents that appear in both rows whereas the stimulus appears
228 in the internal row only. But the membrane mechanisms add membrane currents
229 only to the internal row and stimuli to both rows. Perhaps the additional
230 confusion was not worth the mechanism efficiency gain ( since there are many
231 more membrane (and synaptic) currents than stimulus currents) but at any rate
232 the matrix filling is ordered so that membrane mechanisms are done before
233 the axial currents. This means that only membrane and stimulus currents are present
234 on the internal diagonal and right hand side. However, only stimulus current
235 is present on the external diagonal and right hand side. The difference
236 of course is the membrane current elements by themselves and the extracellular
237 code fills the elements appropriately.
238 So, in the code below, the order for doing the extracellular setup is very
239 important with respect to filling the matrix and mechanisms must be first,
240 then the obsolete activsynapse, then the extracellular setup, then
241 linear models and the obsolete activstim and activclamp. I.e it is crucial
242 to realize that before extracellular has been called that the internal
243 current balance matrix rows are with respect to vm whereas (except for special
244 handling of stimulus terms by keeping separate track of those currents)
245 and after extracellular has been called the current balance matrix rows are
246 with respect to vi and vext.
247 Finally at the end it is possible to add the internal axial terms.
248 */

256 The existing fixed step setup stategy is almost identical to that needed
257 by the current balance portion jacobian solvers for cvode and daspk.
258 However it is wasteful to calculate the right hand side since that is
259 thrown away because those solvers demand a calculation of M*x=b where b
260 is provided and the answer ends up in b. Also, for the function evaluation
261 portion of those solvers, the reuse of the setup is wastful because
262 after setting up the matrix, we essentially calculate rhs - M*x
263 and then throw away M.
264 
265 So, without reducing the efficiency of the fixed step it would be best
266 to re-implement the setup functions to separate the calculation of M
267 from that of b. Those pieces could then be used by all three methods as
268 needed. Also at this level of strategy we can properly take into
269 account cvode's and daspk's requirement of separating the dy/dt terms
270 from the y dependent terms. Note that the problem we are dealing with
271 is entirely confined to the current balance equations and little or
272 no change is required for the mechanism ode's.
273 
274 One other detail must be considered.
275 The computation that memb_func[i].current presently
276 performs can be split into two parts only if the jacabian calculation is done
277 first. This is because the first BREAKPOINT evaluation is for
278 v+.001 and the second is for v. We can't turn that around. The best that
279 can be done (apart from analytic calculation of g) is to make use
280 of the simultaneous calculation of g for the fixed step, store it for
281 use when the jacobian is evalulated and eventually
282 have faster version of current only for use by cvode and daspk.
283 
284 A further remark. LinearMechanisms allow much greater generality in
285 the structure of the current balance equations, especially with regard
286 to extracellular coupling and nonlinear gap junctions. In both these
287 cases, the extracellular equivalent circuit apparatus is entirely
288 superfluous and only the idea of a floating extracellular node is
289 needed. Perhaps this, as well as the elimination of axial resistors
290 for single segment sections will be forthcoming. Therefore it
291 is also useful to keep a clear separation between the computation of
292 membrane currents intra- and extracellular contribution
293 and the calculation of intracellular axial and extracellular currents.

299 cx*dvx/dt - cm*dvm/dt = -gx*(vx - ex) + i(vm) + ax_j*(vx_j - vx)
300 It has advantage of not adding capacitance terms to the

302 for addition to the right hand side, which would have to be done anyway

1474 static double diam_from_list(Section* sec, int inode, Prop* p, double rparent)
1475  /* p->param[0] is diam of inode in sec.*/
1476  /* rparent right half resistance of the parent segment*/
1477 {
1478  /* Basic algorithm assumes a set of monotonic points on which a
1479  function is defined. The extension is the piecewise continuous
1480  function. y(x) for 0<=x<=x[n] (extrapolate for x>x[n]).
1481  The problem is to form the integral of f(y(s)) over intervals
1482  s[i] <= s <= s[i+1]. Note the intervals don't have to be equal.
1483  This implementation is specific to a call to this function for
1484  each interval in order.
1485  */
1486  /* computes diam as average, area, and ri. Slightly weirder since
1487  interval spit in two to compute right and left half values of ri
1488  */
1489  /* fills NODEAREA and NODERINV and returns the right half resistance
1490  (MOhms) of the segment.
1491  */
1492  static int j;
1493  static double x1, y1, ds;
1494  int ihalf;
1495  double si, sip;
1496  double diam, delta, temp, ri, area, ra, rleft=0.0;
1497  int npt, nspine;
1498 
1499  if (inode == 0) {
1500  j = 0;
1501  si = 0.;
1502  x1 = sec->pt3d[j].arc;
1503  y1 = fabs(sec->pt3d[j].d);
1504  ds = sec->pt3d[sec->npt3d - 1].arc / ((double)(sec->nnode - 1));
1505  }
1506  si = (double)inode*ds;
1507  npt = sec->npt3d;
1508  diam = 0.;
1509  area = 0.;
1510  nspine = 0;
1511  ra = nrn_ra(sec);
1512  for (ihalf = 0; ihalf < 2; ihalf++) {
1513  ri = 0.;
1514  sip = si + ds/2.;
1515  for (;;) {
1516  int jp, jnext;
1517  double x2, y2, xj, xjp;
1518  jp = j + 1;
1519  xj = sec->pt3d[j].arc;
1520 #if NTS_SPINE
1521  if (sec->pt3d[j].d < 0 && xj >= si && xj < sip) {
1522  nspine++;
1523  }
1524 #endif
1525  xjp = sec->pt3d[jp].arc;
1526  if (xjp > sip || jp == npt - 1) {
1527  double frac;
1528  if (fabs(xjp - xj) < 1e-10) {
1529  frac = 1;
1530  }else{
1531  frac = (sip - xj)/(xjp - xj);
1532  }
1533  x2 = sip;
1534  y2 = (1. - frac)*fabs(sec->pt3d[j].d) + frac*fabs(sec->pt3d[jp].d);
1535  jnext = j;
1536  }else{
1537  x2 = xjp;
1538  y2 = fabs(sec->pt3d[jp].d);
1539  jnext = jp;
1540  }
1541 
1542  /* f += integrate(x1,y1, x2, y2) */
1543  delta = (x2 - x1);
1544  diam += (y2 + y1)*delta;
1545  if (delta < 1e-15) {
1546  delta = 1e-15;
1547  }
1548  if ((temp = y2*y1/delta) == 0) {
1549  temp = 1e-15;
1550  }
1551  ri += 1/temp;
1552 #if 1 /*taper is very seldom taken into account*/
1553  temp = .5*(y2 - y1);
1554  temp = sqrt(delta*delta + temp*temp);
1555 #else
1556  temp = delta;
1557 #endif
1558 
1559  area += (y2 + y1)*temp;
1560  /* end of integration */
1561 
1562  x1 = x2;
1563  y1 = y2;
1564  if (j == jnext) {
1565  break;
1566  }
1567  j = jnext;
1568  }
1569  if (ihalf == 0) {
1570  rleft = ri*ra/PI*(4.*.01); /*left seg resistance*/
1571  }else{
1572  ri = ri*ra/PI*(4.*.01); /* MegOhms */
1573  /* above is right half segment resistance */
1574  }
1575  si = sip;
1576  }
1577  /* answer for inode is here */
1578  NODERINV(sec->pnode[inode]) = 1./(rparent + rleft);
1579  diam *= .5/ds;
1580  if (fabs(diam - p->param[0]) > 1e-9 || diam < 1e-5) {
1581  p->param[0] = diam; /* microns */
1582  }
1583  NODEAREA(sec->pnode[inode]) = area*.5*PI;/* microns^2 */
1584  UPDATE_VEC_AREA(sec->pnode[inode]);
1585 #if NTS_SPINE
1586  /* if last point has a spine then increment spine count for last node */
1587  if (inode == sec->nnode-2 && sec->pt3d[npt-1].d < 0.) {
1588  nspine += 1;
1589  }
1590  NODEAREA(sec->pnode[inode]) += nspine*spinearea;
1591  UPDATE_VEC_AREA(sec->pnode[inode]);
1592 #endif
1593  return ri;
1594 }
